# MCP Task Flow

本文档定义每个任务从生成到发布的标准流程。

## V3 标准任务流

```text
需求进入
  -> 仓颉检索历史记忆
  -> 伏羲检索知识库
  -> 鬼谷子判断战略影响
  -> 扁鹊登记新知识吸收需求
  -> 诸葛亮查询 Capability Matrix
  -> 赵云预检查 Knowledge Certification
  -> 诸葛亮生成任务
  -> 墨子确认架构边界
  -> 专业 Codex 执行
  -> 赵云审计
  -> 仓颉记录决策、失败或发布记忆
  -> 伏羲回流可复用知识
  -> 扁鹊更新专家训练记录
```

## 任务生成规则

任务必须来自以下来源之一：

1. 已确认 PRD
2. 已确认架构文档
3. 已确认数据治理计划
4. 已确认接口协议
5. 已确认审计整改项
6. 已确认知识库缺口
7. 已确认记忆中心历史问题
8. King Xu 或女娲直接指令
9. 赵云发起的审计整改任务

## 任务拆分规则

任务拆分时必须满足：

1. 单任务有明确责任 Codex。
2. 单任务输出可审计。
3. 单任务依赖关系清楚。
4. 单任务验收标准可验证。
5. 单任务失败不会模糊影响范围。
6. 命中专家条件时必须添加专家协作角色。
7. 命中专家条件时必须检查专家训练状态。
8. 命中专家条件时必须检查专家成熟度 Level。
9. 命中专家条件时必须检查 Capability Matrix。
10. 命中专业知识域时必须检查 Knowledge Certification。
11. AOEM 任务必须检查 AOEM Constitution。
12. 任务必须记录知识库引用、全局记忆引用和项目记忆引用。

## 专家角色触发规则

任务拆分时如命中以下条件，诸葛亮必须把对应专家加入协作 Codex：

| 专家 | 触发条件 | 训练门禁 | 必须验证 |
|---|---|---|---|
| 庞统 / AOEM-001 | AOEM 内核、GPU 调度、隐私计算、代数执行、运行时优化 | 必须 TRAINED，Level 和认证满足任务边界 | AOEM Constitution、Capability Matrix、Knowledge Certification |
| 华佗 / MATH-001 | 代数结构、数学证明、索尔算法、量子计算、模型数理验证 | 必须 TRAINED，Level 和认证满足任务边界 | Capability Matrix、Knowledge Certification、math-model-audit |
| 庄周 / LANG-001 | 新语言语义、DSL、指令集、编译优化、自动演化 | 必须 TRAINED，Level 和认证满足任务边界 | Capability Matrix、Knowledge Certification、墨子审批 |

## 任务依赖图规则

任务依赖图应按以下顺序生成：

1. 资料吸收任务
2. 知识库建设任务
3. 记忆检索任务
4. 战略判断任务
5. 文档任务
6. 架构任务
7. 数据任务
8. 接口任务
9. UI 任务
10. 后端任务
11. 前端任务
12. 移动端任务
13. AI 任务
14. AOEM 专项任务
15. 数学模型任务
16. 新语言任务
17. 测试任务
18. 审计任务
19. 发布任务
20. 记忆归档任务

## FAIL 处理规则

审计失败必须给出：

1. 失败原因
2. 风险级别
3. 影响范围
4. 整改建议
5. 二次复审标准
6. 是否阻断发布
7. 是否记录失败经验

## 发布前确认

发布前必须确认：

1. 任务卡状态为 `AUDIT_PASS`
2. 相关文档已经更新
3. 知识库引用有效
4. 记忆检索已完成
5. Capability Matrix 检查通过
6. Knowledge Certification 检查通过
7. AOEM 任务未违反 AOEM Constitution
8. 专家训练状态满足输出要求
9. 测试结果可追溯
10. 审计报告已归档
11. 发布与回滚负责人明确
12. 项目经验回流路径已确认
13. 命中人工确认红线的任务已经得到 King Xu 确认
