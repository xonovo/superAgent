# Language Evolution Spec

## 1. 定位

新语言自动演化用于为 OMCF 项目提供 DSL、指令集、编译优化和 AI Agent 指令抽象能力。

## 2. 语言方案必须包含

1. 目标。
2. 非目标。
3. 语义模型。
4. 类型或数据边界。
5. 指令集。
6. 错误处理。
7. 权限边界。
8. 编译或解释策略。
9. 与 AOEM、AI、后端系统的交互方式。

## 3. 自动演化原则

1. 每次演化必须有版本号。
2. 每次演化必须保留变更原因。
3. 每次演化必须提供兼容性说明。
4. 自动演化不得绕过墨子审批。
5. 自动演化不得绕过赵云审计。

## 4. 禁止事项

1. 未定义语义就生成实现。
2. 未经过 compiler-sanity-test 就进入主链路。
3. 以优化为由改变业务语义。
4. 以自动演化为由绕过权限或审计。
