# 发布规则

## 发布前条件

1. 任务状态为 `AUDIT_PASS`。
2. 文档已更新。
3. 测试结果已归档。
4. 回滚方案已验证。
5. 生产影响范围已说明。
6. 发布窗口已确认。
7. 发布负责人和回滚负责人已明确。

## 禁止发布条件

出现以下情况不得发布：

1. 审计未通过。
2. 数据迁移不可回滚。
3. 外部接口对账规则不清。
4. 权限风险未解决。
5. 测试结果缺失。
6. 生产凭据或敏感信息暴露。

## 发布后检查

1. 服务健康状态。
2. 关键接口可用性。
3. 数据库连接和性能。
4. 外部接口回调。
5. 审计日志写入。
6. 错误日志和告警。
