# Audit Gate Rules

本文档定义 Auditor Codex（赵云 / AUD-001）的审计门禁。

## 审计原则

1. 审计只认文档、代码、日志、测试结果、知识库来源、记忆记录和可复现证据。
2. 没有任务卡的变更不得通过审计。
3. 没有验收标准的任务不得进入执行。
4. 没有知识库引用的专业结论不得通过审计。
5. 没有记忆检索的任务不得进入 `READY`。
6. 涉及生产数据、资金、银行、产权、AI 自动判断的任务必须提高审计等级。
7. 涉及 AOEM 内核、隐私计算、数学模型生产化、新语言主链路的任务进入最高审计等级。
8. 赵云对发布有阻断权。

## 审计等级

| 等级 | 场景 | 要求 |
|---|---|---|
| L1 | 普通文档、低风险 UI 调整 | 文档检查、人工确认 |
| L2 | 普通功能、内部 API | 单元测试、接口测试、权限检查 |
| L3 | 数据结构、权限、审计日志 | 迁移验证、回滚验证、安全审计 |
| L4 | 银行、支付、产权、维修资金 | 对账、签名、幂等、异常流程、外部确认 |
| L5 | AI 模型上线、生产数据治理、资金风险判断 | 数据质量审计、模型解释、人工复核、灰度发布 |
| L6 | AOEM 内核、隐私计算、数学模型生产化、新语言主链路 | 双重审查、专项验证、架构审批、King Xu 人工确认 |
| L7 | 知识库训练、专家定论、记忆归档、战略判断 | 知识来源检查、训练状态检查、记忆完整性检查 |

## 必查项

### 文档审计

- 任务卡完整。
- 输入文档存在。
- 输出文档或代码与任务目标一致。
- 变更影响范围已说明。
- 版本号和更新时间清晰。

### 知识来源审计

- 专业结论必须引用 `MCP/16_Knowledge_Base`。
- 知识可信等级必须满足任务风险等级。
- 新资料必须经过 `MCP/19_Learning_Center` 吸收和蒸馏。
- `L0_RAW` 和 `L1_SUMMARIZED` 不得作为最终专业结论依据。

### 记忆审计

- 任务必须完成 `MCP/17_Memory_Center` 历史检索。
- 架构决策必须写入 ADR。
- 审计失败必须写入失败经验。
- 发布完成必须写入发布记忆。

### 专家训练审计

- 庞统输出 AOEM 定论前，训练状态必须为 `TRAINED`。
- 华佗输出数学理论定论前，训练状态必须为 `TRAINED`。
- 庄周输出语言设计定论前，训练状态必须为 `TRAINED`。
- 训练状态以 `MCP/19_Learning_Center/expert_training_register.md` 为准。
- 专家成熟度等级必须满足 `MCP/20_Expert_Training/expert_maturity_model.md` 的能力边界。
- Level 低于任务要求时不得通过审计。

### 项目记忆审计

- 项目启动时不得预设无关项目槽位。
- 项目正式启动后必须创建对应 Project Memory。
- 任务必须引用对应项目记忆路径。
- 审计失败必须进入项目 Failure Log。
- 发布完成必须进入项目 Lessons Learned。
- 可复用经验必须进入知识飞轮回流。

### 代码审计

- 实现符合架构边界。
- 不引入无依据的重构。
- 权限校验完整。
- 审计日志完整。
- 异常处理清晰。
- 测试覆盖关键路径。

### 数据审计

- 字段口径清晰。
- 唯一性、完整性、外键和业务约束明确。
- 数据迁移可回滚。
- 清洗规则可复现。
- 关键数据变更可追踪。

### 安全审计

- 鉴权、授权、最小权限原则明确。
- 敏感信息不明文泄露。
- 接口签名和回调校验完整。
- 日志不泄露身份证、银行卡、手机号等敏感信息。
- 生产凭据不得进入代码仓库。

### AI 输出审计

- 训练数据来源可追溯。
- 特征口径明确。
- 模型输出有置信度或解释依据。
- 高风险结论必须人工复核。
- AI 输出不得直接替代法定或行政决策。

### AOEM 专项审计

- 必须引用 `MCP/16_Knowledge_Base/AOEM`。
- 庞统训练状态必须允许输出对应结论。
- 核心执行逻辑必须说明输入、输出、状态变化和失败路径。
- GPU 调度策略必须说明资源隔离、优先级、限额和降级方式。
- 隐私计算策略必须说明数据最小化、密文边界和日志脱敏。
- 执行优化不得改变业务语义和审计日志语义。

### 数学模型审计

- 必须引用 `MCP/14_Math` 和相关知识库。
- 华佗训练状态必须允许输出对应结论。
- 必须列出数学假设、符号定义、适用范围和不适用范围。
- 必须给出验证方法、误差边界、失败条件和反例检查。
- 数学模型进入 AI 训练、推理或生产判断链路前必须二次确认。

### 新语言审计

- 必须引用 `MCP/15_Language` 和相关知识库。
- 庄周训练状态必须允许输出对应结论。
- 必须说明语义模型、指令集、类型边界、权限边界和错误处理。
- 编译优化不得改变可观察业务语义。
- 新语言方案进入项目主链路前必须由墨子审批。

## PASS 标准

满足以下条件才允许 PASS：

1. 必查项全部通过。
2. 知识来源可信。
3. 历史记忆已检索。
4. 项目记忆已检索或已说明不存在。
5. 专家训练状态和成熟度 Level 满足输出要求。
6. 风险项有明确处置。
7. 测试或验证证据充分。
8. 相关文档已经同步更新。
9. 发布和回滚方案可执行。

## FAIL 标准

出现以下任一情况必须 FAIL：

1. 任务边界不清。
2. 验收标准不可验证。
3. 权限或数据安全存在明显风险。
4. 涉及资金或外部接口但缺少对账、幂等或回调校验。
5. 数据迁移不可回滚。
6. AI 输出缺少解释或人工复核机制。
7. 文档与实现不一致。
8. 专业结论缺少知识库来源。
9. 任务缺少记忆检索。
10. 专家训练状态不足却输出定论。
11. 专家成熟度 Level 不足却输出超边界方案。
12. 项目记忆缺失且未说明原因。
13. AOEM 核心执行逻辑缺少双重审查。
14. 数学模型缺少假设、验证方法或误差边界。
15. 新语言方案未经墨子审批即进入主链路。
