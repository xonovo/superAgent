# GPU Dispatch Policy

## 1. 调度目标

GPU 调度必须同时满足：

1. 资源可控。
2. 任务可追踪。
3. 高优先级任务可保障。
4. 低优先级任务可降级。
5. 异常时可回收资源。

## 2. 调度维度

1. 项目优先级。
2. 任务类型。
3. 显存需求。
4. 推理或训练时长。
5. 数据敏感等级。
6. 是否允许降级到 CPU。

## 3. 必须记录

1. 任务编号。
2. 调用角色。
3. 模型或内核版本。
4. GPU 资源占用。
5. 起止时间。
6. 失败原因。
7. 降级路径。

## 4. 风险控制

生产任务不得与实验任务共享未隔离的 GPU 执行上下文。涉及敏感数据的任务必须先通过数据治理和隐私计算边界检查。
