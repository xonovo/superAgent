# 文档标准

## 文档编号

建议格式：

```text
SUPERAGENT-MCP-领域-序号-版本
```

示例：

```text
SUPERAGENT-MCP-API-001-V0.1
SUPERAGENT-MCP-AUDIT-001-V0.1
```

## 文档最低要求

每份正式文档必须包含：

1. 标题
2. 版本
3. 状态
4. 责任角色
5. 更新时间
6. 适用范围
7. 正文
8. 审计或验收标准

## 文档归档规则

1. 只有 `APPROVED` 和 `ACTIVE` 文档可以作为开发依据。
2. `DRAFT` 文档可以用于讨论，但不得直接触发编码。
3. 重大变更必须保留历史版本说明。
4. 招投标、汇报、审计材料必须与 MCP 事实一致。
