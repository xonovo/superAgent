# 数据治理计划

## 治理对象

首批治理对象为 66 万房屋数据，并向人、房、楼栋、小区、物业企业、收费、银行流水、产权登记、维修资金等数据域扩展。

## 治理目标

1. 数据可追溯。
2. 字段口径统一。
3. 重复数据可识别。
4. 缺失数据可定位。
5. 异常数据可解释。
6. 产权同步可校验。
7. 银行同步可校验。
8. AI 训练数据可审计。

## 治理流程

```text
原始数据接入
  -> 字段映射
    -> 质量检查
      -> 重复识别
        -> 异常标记
          -> 人工复核
            -> 清洗入库
              -> 数据质量报告
                -> AI 训练数据准备
```

## 数据质量维度

1. 完整性
2. 唯一性
3. 一致性
4. 准确性
5. 时效性
6. 可追溯性
7. 安全等级
8. AI 可用性

## 首批检查项

1. 房屋唯一编号是否存在。
2. 小区、楼栋、单元、房号层级是否完整。
3. 建筑面积、套内面积、用途字段是否异常。
4. 产权人或关联人是否缺失。
5. 同一房屋是否重复。
6. 同一产权人与多房屋关系是否合理。
7. 产权登记数据与平台房屋数据是否一致。
8. 银行缴费数据与收费台账是否一致。
9. 敏感字段是否脱敏或分级。
10. 字段是否可用于 AI 特征。

## AI 训练数据前置条件

训练数据必须满足：

1. 来源明确。
2. 清洗规则明确。
3. 质量检查通过。
4. 敏感信息处理符合安全规则。
5. 特征口径经过 AI Codex 和 Data Governance Codex 双方确认。
6. 高风险模型训练集必须经过 Auditor Codex 审计。
