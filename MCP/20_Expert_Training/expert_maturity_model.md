# Expert Maturity Model

## 总体等级

| Level | 名称 | 能力边界 | 输出限制 |
|---|---|---|---|
| Level 0 | 未训练 | 不理解该领域知识 | 只能提出资料缺口 |
| Level 1 | 了解文档 | 能复述基础资料 | 只能输出摘要和问题 |
| Level 2 | 理解机制 | 能解释运行机制和边界 | 可输出分析，不得独立设计 |
| Level 3 | 能独立开发 | 能在既有规范内实现 | 可输出开发方案，需审计 |
| Level 4 | 能独立优化 | 能发现瓶颈并优化 | 可输出优化方案，需复核 |
| Level 5 | 能设计新能力 | 能提出新架构、新理论、新语言 | 必须 King Xu 或女娲确认 |

## 庞统 / AOEM 专家

| Level | 能力标准 |
|---|---|
| Level 0 | 未训练 |
| Level 1 | 了解 AOEM 文档 |
| Level 2 | 理解 AOEM Runtime、State、Execution |
| Level 3 | 能独立开发 AOEM 扩展 |
| Level 4 | 能独立优化 AOEM Runtime、GPU、Privacy |
| Level 5 | 能设计 AOEM 新能力 |

## 华佗 / 数学专家

| Level | 能力标准 |
|---|---|

## 升级要求

每次升级必须具备：

1. 知识库来源。
2. 训练任务记录。
3. 练习或项目应用证据。
4. 审计结果。
5. 成熟度评估记录。
