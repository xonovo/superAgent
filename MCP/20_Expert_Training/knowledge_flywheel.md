# Knowledge Flywheel

## 目标

知识飞轮用于让 OMCF 在每个项目后变得更强，而不是每次重新开始。

## 标准闭环

```text
资料进入
  -> 扁鹊吸收
    -> 伏羲结构化
      -> 专家训练
        -> 项目应用
          -> 赵云审计
            -> 仓颉沉淀经验
              -> 伏羲回流知识库
                -> 扁鹊更新训练集
```

## 角色分工

| 环节 | 责任角色 | 输出 |
|---|---|---|
| 资料进入 | 扁鹊 | 学习吸收记录 |
| 知识整理 | 伏羲 | 结构化知识、可信等级 |
| 专家训练 | 扁鹊 + 专家 | 训练记录、成熟度评估 |
| 项目应用 | 专业 Codex | 任务产物 |
| 审计验证 | 赵云 | 审计报告 |
| 经验沉淀 | 仓颉 | 决策、失败、发布、经验记录 |
| 知识回流 | 伏羲 | 知识库更新 |

## 回流触发条件

以下情况必须触发经验回流：

1. 任务审计失败。
2. 二次复审失败。
3. 生产发布完成。
4. 出现高价值技术方案。
5. 出现可复用业务规则。
6. 专家成熟度升级。

## 禁止事项

1. 项目经验不得只留在聊天上下文里。
2. 审计失败不得只给整改建议，必须进入失败经验。
3. 新知识不得绕过扁鹊直接成为专家结论。
4. 未回流知识库的经验不得作为跨项目复用依据。
