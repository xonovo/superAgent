# 架构原则

## 总体原则

1. 监管、数据、资金、AI、审计分层清晰。
2. 核心业务服务不得与外部接口适配逻辑混杂。
3. 数据治理能力独立成体系。
4. 审计日志贯穿所有关键操作。
5. AI 输出必须与业务系统解耦，避免模型结果直接支配行政或资金决策。

## 建议分层

```text
UI 层
  -> 应用服务层
    -> 领域服务层
      -> 数据访问层
        -> 数据库和外部系统

AI 服务层
  -> 数据蒸馏
    -> 特征服务
      -> 推理服务
        -> 人工复核和审计
```

## 模块边界

建议优先划分：

1. 组织与权限
2. 房屋与产权
3. 小区与物业企业
4. 收费与资金
5. 公共收益
6. 维修资金
7. 外部接口
8. 审计日志
9. 数据治理
10. AI 画像与风险
11. 运维与发布

## 禁止事项

1. 未定义数据口径前创建核心表。
2. 未定义权限规则前实现关键业务操作。
3. 未定义审计日志前接入资金相关接口。
4. 未完成数据治理前训练生产模型。
