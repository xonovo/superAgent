# OMCF V2.1 Master Control Protocol

## 1. 定位

MCP V2.1 是 superAgent OMCF 的项目总控协议，用于管理 AI Software Factory 的组织结构、任务流、文档库、审计规则和发布规则。

它不是一个单纯的代码目录，而是 AI Agent 集群的工作宪法。`MCP` 在本文档中表示 `Master Control Protocol`，不表示 Anthropic 的 `Model Context Protocol`。

## 2. 管理链路

```text
King Xu
  -> 女娲 / CAIO
    -> 诸葛亮 / PM-001
      -> 墨子 / ARC-001
        -> 专业 Codex Agent
          -> 赵云 / AUD-001
            -> 发布、退回或人工确认
```

## 3. 复用模型

OMCF 分为四层：

1. `OMCF-Core`：永久组织、角色矩阵、协作原则。
2. `OMCF-Project-Pack`：项目名称、项目资料、业务规则、现有系统材料。
3. `OMCF-Delivery-System`：任务拆解、文档建设、设计、开发、测试、部署。
4. `OMCF-Audit-Gate`：文档审计、代码审计、数据审计、安全审计、发布审计。

任何新项目只需要替换 `Project-Pack`，组织和流程不变。

## 6. 任务生命周期

```text
需求进入
  -> 诸葛亮生成任务
  -> 女娲确认组织与规则适配
  -> 墨子确认架构边界
  -> MCP 分配专业 Codex
  -> 专业 Codex 执行
  -> 赵云审计
  -> PASS 发布
  -> FAIL 进入二次复审
  -> 二次复审 PASS 发布
  -> 二次复审 FAIL 退回重新执行
```

## 7. 任务状态

每个任务必须处于下列状态之一：

- `DRAFT`：草稿，尚未进入执行。
- `READY`：任务边界、输入、输出、审计标准已明确。
- `ASSIGNED`：已分配给专业 Codex。
- `IN_PROGRESS`：执行中。
- `SUBMITTED`：已提交审计。
- `AUDIT_PASS`：审计通过。
- `AUDIT_FAIL`：审计失败。
- `SECOND_REVIEW`：二次复审中。
- `REWORK`：返工中。
- `HUMAN_CONFIRM_REQUIRED`：需要 King Xu 人工确认。
- `RELEASED`：已发布。
- `ARCHIVED`：已归档。

## 9. 专家角色触发规则

### 庞统 / AOEM-001

出现下列情况必须引入庞统：

1. AOEM 内核、运行时、执行引擎或扩展能力。
2. GPU 调度、隐私计算、代数执行、执行优化。
3. AI 推理链路需要内核级性能优化。
4. 需要生成 AOEM 内核相关文档或验证方案。

庞统输出核心执行逻辑必须经过赵云审计，并由墨子或张飞进行技术复核。

### 华佗 / MATH-001

出现下列情况必须引入华佗：

1. 代数结构、代数语义、数学证明、误差边界。
2. 索尔算法、量子计算、复杂优化算法。
3. AI 模型数理验证、数据蒸馏数学假设。
4. 需要证明模型、规则或算法的有效性。

华佗输出数学模型必须提供假设、推导、验证方法和适用边界，并进入二次确认。

### 庄周 / LANG-001

出现下列情况必须引入庄周：

1. 新语言语义、DSL、指令集设计。
2. 自动编译优化、自动语言演化。
3. AOEM 指令扩展或 AI Agent 指令抽象。
4. 需要生成新语言原型或语言规范。

庄周输出的新语言方案必须由墨子审批后才能进入项目主链路。

## 10. 文档优先规则

涉及下列内容时，必须先产出或更新文档，再允许编码：

1. 业务流程变化
2. 数据结构变化
3. 权限规则变化
4. 外部接口变化
5. 审计规则变化
6. 发布流程变化
7. AI 模型训练或推理规则变化
8. AOEM 内核、GPU 调度或隐私计算变化
9. 数学模型、代数语义或量子算法变化
10. 新语言语义、指令集或编译器变化
11. 安全策略变化

## 11. 审计门禁

任务至少经过以下门禁：

1. 文档完整性审计
2. 业务一致性审计
3. 架构边界审计
4. 数据安全审计
5. 代码质量审计
6. 权限与审计日志审计
7. 外部接口审计
8. AI 输出审计
9. AOEM 核心逻辑审计
10. 数学模型验证审计
11. 新语言语义审计
12. 测试覆盖审计
13. 发布风险审计

任何关键门禁失败，任务不得发布。

## 12. 发布原则

发布必须满足：

1. 任务状态为 `AUDIT_PASS`
2. 关键文档已更新
3. 自动化测试通过
4. 数据迁移方案可回滚
5. 外部接口变更已确认
6. AI、AOEM、数学模型或新语言专项变更已完成专项审计
7. 审计日志可追踪
8. 发布窗口和回滚负责人明确

## 13. 版本规则

MCP 文档采用语义化版本：

- `V0.x`：草案
- `V1.x`：初版可执行规则
- `V2.x`：纳入 AI Agent 集群治理
- `V2.1`：纳入 AOEM、数学、新语言专家角色
- `V3.x`：纳入生产级审计和自动发布

当前版本：`V2.1-DRAFT`

3. 任务流和审计流可执行。
4. 产品、架构、数据、API、AI、AOEM、数学、新语言、测试、部署都有初版规范。
