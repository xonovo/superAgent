# 阶段划分

## 版本信息

- 版本：V0.3
- 状态：DRAFT
- 责任角色：诸葛亮 / PM-001

## 第一阶段：MCP 文档化与记忆初始化

目标：

1. 建立 MCP 目录结构。
2. 建立角色矩阵。
3. 建立任务流、审计流和发布流。
4. 建立产品、架构、数据、API、UI、AI、AOEM、数学、新语言、测试、部署的初版规范。
5. 建立知识库、记忆中心、战略中心、学习中心。
6. 建立专家训练状态登记。

## 第二阶段：任务计划生成

目标：

1. 生成任务树。
2. 生成任务依赖图。
3. 生成里程碑。
4. 生成开发计划。
5. 生成风险清单。
6. 生成知识库缺口清单。
7. 生成记忆检索报告。
8. 识别需要庞统、华佗、庄周参与的专家任务。

## 第三阶段：受控开发

目标：

1. 按任务卡执行开发。
2. 按审计门禁检查。
3. 按发布规则上线。
4. 涉及人工确认红线时暂停执行并等待 King Xu 确认。
5. 所有失败、返工和发布结果进入记忆中心。

## 第四阶段：AI、AOEM 与专家能力接入

目标：

1. 建立数据蒸馏引擎。
2. 建立画像模型。
3. 建立风险识别和预测模型。
4. 接入 GPU 推理集群。
5. 接入 AOEM 内核调度、数学模型验证和新语言指令抽象。
6. 庞统、华佗、庄周达到对应知识域 `TRAINED` 状态后再输出专业定论。
