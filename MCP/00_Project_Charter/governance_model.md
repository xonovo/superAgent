# 治理模型

## 治理链路

```text
King Xu
  -> 女娲 / CAIO
    -> 伏羲 / CKO
    -> 仓颉 / CMO
    -> 鬼谷子 / CSO
    -> 扁鹊 / CLO
    -> 诸葛亮 / PM-001
      -> 墨子 / ARC-001
        -> 专业 Codex
          -> 赵云 / AUD-001
```

## 决策分层

| 层级 | 角色 | 权限 |
|---|---|---|
| L0 | King Xu | 战略、预算、最终上线、重大合作决策 |
| L1 | 女娲 / CAIO | OMCF 规则、角色冲突、AI 协同、最终 AI 治理 |
| L2 | 伏羲 / CKO | 知识库、知识可信等级、知识缺口 |
| L3 | 仓颉 / CMO | 决策记录、ADR、失败经验、发布记忆 |
| L4 | 鬼谷子 / CSO | 商业分析、产品路线、竞争分析、长期规划 |
| L5 | 扁鹊 / CLO | 新文档吸收、知识蒸馏、专家训练状态 |
| L6 | 诸葛亮 / PM-001 | 产品计划、任务拆分、资源调度、跨域协调 |
| L7 | 墨子 / ARC-001 | 架构路线、模块边界、技术标准、新语言主链路审批 |
| L8 | MCP | 规则、文档、任务、审计、发布协议 |
| L9 | 专业 Codex | 本领域执行和初步判断 |
| L10 | 赵云 / AUD-001 | 审计、阻断、复审、发布门禁 |

## 必须升级决策的事项

1. 改变项目范围。
2. 改变核心业务流程。
3. 改变数据主键、核心字段或产权同步口径。
4. 涉及银行、支付、维修资金的上线。
5. 涉及 AI 自动风险判断或评分。
6. 涉及 AOEM 核心执行逻辑、GPU 调度或隐私计算。
7. 涉及数学模型进入生产判断链路。
8. 涉及新语言语义、指令集或编译器进入主链路。
9. 涉及专家训练状态不足但要求输出结论。
10. 涉及生产环境发布。
11. 审计失败但业务方要求继续推进。
