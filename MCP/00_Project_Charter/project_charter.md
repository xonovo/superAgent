# 项目章程

## 项目名称

superAgent OMCF / WPF-MCP V4。

## 项目定位

本项目面向住建局、城市物业公司、银行、审计、物业企业、业主及相关监管单位，建设一个可监管、可审计、可扩展、可接入 AI 能力的物业领域平台。

同时，本项目建设 OMCF（OneMan AI Company Framework），使 King Xu 可以通过女娲、伏羲、仓颉、鬼谷子、扁鹊、诸葛亮、墨子、赵云和各专业 Codex，在统一文档、统一规则、统一知识、统一记忆、统一审计体系下复用到更多项目。

## 总体目标

1. 建立物业监管平台的业务、数据、接口、AI、审计和部署标准。
2. 建立 66 万房屋数据治理体系。
3. 建立人、房屋、小区、物业企业、收费、公共收益和风险画像体系。
4. 建立可审计的资金、产权、维修资金和外部接口链路。
5. 建立 OMCF 驱动的多 Codex 协作研发体系。
6. 建立 AOEM、数学建模、新语言演化三类专家协作能力。
7. 建立知识库、记忆中心、战略中心和学习中心。
8. 建立专家成熟度、知识飞轮、项目记忆和经验回流体系。
9. 建立能力矩阵、知识认证、AOEM 宪法和决策注册表。

## 项目原则

1. 文档先于代码。
2. 数据口径先于模型。
3. 审计先于发布。
4. 权限先于功能。
5. 可回滚先于上线。
6. 人工复核先于 AI 自动决策。
7. 内核、数学、新语言专项能力先训练后结论。
8. 专业结论必须有知识库依据。
9. 历史判断必须先检索记忆中心。
10. 项目记忆必须按需创建，不得预设项目槽位。
11. 任务分配必须先查能力矩阵和知识认证。

## 一期范围

一期聚焦：

1. MCP 文档体系。
2. 产品和业务流程梳理。
3. 系统架构和模块拆分。
4. 核心数据模型和 66 万房屋数据治理规则。
5. API 和外部接口标准。
6. 审计门禁和发布规则。
7. AOEM、数学、新语言专家规范。
8. 知识库和专家训练登记。
9. 记忆中心、决策记录和失败经验机制。
10. 战略中心和学习中心机制。
11. 专家成熟度和知识飞轮机制。
12. 能力矩阵、知识认证、AOEM 宪法和决策注册表。
13. 任务树、依赖图、里程碑、开发计划和风险清单。

## 暂不进入一期的内容

1. 大规模编码。
2. 生产级 AI 模型上线。
3. 未经治理的数据训练。
4. 未经审计的外部资金接口上线。
5. 未经确认的生产数据迁移。
6. 未经训练的 AOEM 核心执行逻辑定论。
7. 未经训练和审批的新语言方案进入主链路。

## 成功标准

第一阶段成功标准：

1. MCP 目录和核心文档完整。
2. 女娲、伏羲、仓颉、鬼谷子、扁鹊和全部项目执行 Codex 角色职责明确。
3. 任务流、审计流、发布流可以执行。
4. 知识库、记忆中心、战略中心、学习中心可以被任务卡引用。
5. 庞统、华佗、庄周的训练状态、成熟度 Level 和输出限制明确。
6. 项目记忆模板可复用，且不预设具体项目槽位。
7. 能力矩阵和知识认证可以支撑任务分配。
8. 后续 Codex 可以根据 MCP 自动生成任务树、开发计划、知识缺口和记忆检索报告。
