使用教程

OMCF 是一人AI公司框架，OMC-OS 是未来的一人AI公司操作系统。按顺序使用：先看状态，再启动任务，再过门禁，最后让 Worker 执行低风险文档任务。

0. 先理解 OMCF / OMC-OS

OMCFOneMan AI Company Framework，一人AI公司框架

OMC-OSOneMan AI Company Operating System，一人AI公司操作系统

现在你看到的是 OMCF Dashboard，也就是未来 OMC-OS 的控制台层。

项目怎么复用

Workspace工作区，管理多个项目

Project项目，每个项目有独立资料、记忆、任务、输出、审计

Agent Pool智能体池，女娲、诸葛亮、墨子、赵云等角色是公司级共享资源

不提前创建空项目槽位。只有你点“新建项目”后，才创建项目目录。

Agent 如何绑定项目

同一个 Agent，绑定不同项目资料后，就进入不同项目状态。

  • 诸葛亮 + 株洲物业资料 = 株洲项目经理
  • 诸葛亮 + NOVOVM资料 = NOVOVM项目经理

角色共享，项目记忆隔离，避免知识污染。

工作台怎么用

工作台OMC-OS 的 IDE 外壳第一版

左边看 Workspace 和新项目申请，右边看真实项目、Agent Pool 和当前项目绑定。

“记录新项目申请”只写 Alpha 日志，不会创建空项目目录。

1. 看公司是否在运行

先看顶部四个数字。

  • Running Agents正在工作的AI角色数量
  • Safe Ready已经通过门禁、可以进入安全队列的命令
  • Commands / Queue命令总数 / 等待Worker处理的数量
  • Worker DoneWorker已经完成的低风险任务

2. 启动一个任务

点顶部“启动下一任务”，或进入“任务流”点某一行的“启动”。系统会创建命令，但不会直接改文件、数据库或生产。

命令创建后会自动进入 Dry Run试跑风险分类。

3. 看门禁结果

点“查看命令日志”，看每条命令的门禁。

  • Dry Run判断任务风险
  • King Xu Approval高风险任务需要你批准
  • Zhao Yun Audit代码或高风险任务需要赵云审计
  • Production Whitelist生产默认禁止，必须手工白名单

4. 安全执行

命令显示 READY_FOR_SAFE_EXECUTION可安全执行 时，才能点“安全执行”。

点完只会进入 Safe Execution Queue安全执行队列，还不是碰生产。

5. Worker 执行

只有低风险文档任务可以点 Worker 执行。

Worker 会调用 Codex 的 read-only 沙箱生成文档，并写入项目记忆，再生成赵云审计记录。

永远不要期待这些按钮自动做的事

  • 不会改数据库结构
  • 不会连接银行或支付接口
  • 不会执行 AOEM 核心能力
  • 不会生产部署
  • 不会删除文件
